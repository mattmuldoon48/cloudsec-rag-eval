# Incident Response Basics (sample)

This sample document describes a cloud incident response process. It is illustrative and not official guidance.

## Response Plan Elements

- Define roles and communication channels.
- Create a clear escalation path.
- Capture evidence and preserve logs.

## Detection and Triage

- Confirm the incident, then classify severity.
- Identify impacted systems and data.
- Block unauthorized access and apply containment.

## Post-Incident Activities

- Conduct a root cause analysis.
- Document lessons learned.
- Update policies and runbooks.
